# WareManager - Frontend Code Analysis Report

## Project Overview
WareManager is a comprehensive stock management web application built with React, TypeScript, and Vite. The application provides a complete solution for managing inventory, warehouses, products, orders, purchases, shipments, receptions, transfers, clients, and suppliers.

## Technology Stack
- **Framework**: React with TypeScript
- **Build Tool**: Vite
- **UI Component Library**: Shadcn UI (built on Radix UI primitives)
- **Styling**: Tailwind CSS
- **State Management & Data Fetching**: React Query (TanStack Query)
- **HTTP Client**: Axios
- **Form Handling**: React Hook Form with Zod validation
- **Routing**: React Router
- **Icons**: Lucide React
- **Charts**: Recharts
- **Notifications**: Sonner

## Project Structure
The project follows a typical React application structure:

- **src/components**: Reusable UI components
  - **src/components/ui**: Primitive UI components from Shadcn UI
- **src/pages**: Page components for different routes
- **src/hooks**: Custom React hooks
- **src/lib**: Utility functions and API services
- **public**: Static assets

## Pages Analysis

### Dashboard
The Dashboard page likely provides an overview of the system with key metrics and charts.

### LoginPage
The LoginPage handles user authentication with a login form.

### Management Pages
The application includes management pages for all entities:
- UserManagement: For managing users and permissions
- WarehouseManagement: For managing warehouses
- ProductManagement: For managing products
- ClientManagement: For managing clients
- SupplierManagement: For managing suppliers
- OrderManagement: For managing customer orders
- PurchaseManagement: For managing purchase orders from suppliers
- ShipmentManagement: For managing outgoing shipments
- ReceptionManagement: For managing incoming receptions
- TransferManagement: For managing transfers between warehouses
- StockManagement: For managing and viewing stock levels

### StockManagement
The StockManagement page is particularly sophisticated with:
- Paginated table for displaying stock data
- Multiple filtering options (all, warehouse, product, low-stock, advanced)
- Search functionality
- Filter dropdowns for warehouses
- URL parameter support for filtering by warehouse or product

## API Services
The application communicates with a backend API using Axios. The API services are organized by entity:
- userApi: CRUD operations for users
- warehouseApi: CRUD operations for warehouses
- orderApi: CRUD operations for orders
- purchaseApi: CRUD operations for purchases
- shipmentApi: CRUD operations for shipments
- receptionApi: CRUD operations for receptions
- clientApi: CRUD operations for clients
- supplierApi: CRUD operations for suppliers
- unitApi: CRUD operations for units
- stockApi: Operations for stock management
- productApi: CRUD operations for products
- transferApi: CRUD operations for transfers

## Custom Hooks
- use-mobile: For detecting mobile devices
- use-toast: For displaying toast notifications
